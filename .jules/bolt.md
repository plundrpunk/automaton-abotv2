## 2024-05-18 - LLM Request Payload Optimization
**Learning:** In the core LLM execution loop (`runtime.rs`), the `ToolCompletionRequest` struct was taking ownership of `Vec<serde_json::Value>` for `messages` and `tools`. Because the loop might iterate multiple times to handle intermediate tool calls, passing ownership forced the code to call `.clone()` on these large, deeply nested JSON arrays before every API request. This introduced significant memory allocation overhead.
**Action:** When designing structs specifically for Serde JSON serialization (especially API request models), always prefer borrowed types (`&[T]`, `&str`) over owned types (`Vec`, `String`). This allows the serializer to read the data without forcing the caller to heap-allocate clones of the underlying structure.
